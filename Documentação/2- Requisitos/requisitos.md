##  Requisitos Funcionais (RF)
Os requisitos funcionais (RF) detalham as funcionalidades essenciais que o sistema deve executar para atender aos objetivos do projeto. A seguir, cada requisito funcional é descrito com uma explicação objetiva sobre sua finalidade no sistema.


| Código | Nome | Descrição |
|--------|------|-----------|
| RF01 | Cadastrar Item de Lixo Verde | Cidadão informa item, localização e imagem |
| RF02 | Localização Automática | Uso de GPS para registro do ponto exato |
| RF03 | Agendar Coleta | Escolha de data/rota conforme cronograma |
| RF04 | Enviar Denúncia | Anexo de foto e localização |
| RF05 | Enviar Imagens | Auxilia identificação do volume do lixo |
| RF06 | Login e Níveis de Acesso | Acesso diferenciado por tipo de usuário |
| RF07 | Mapa por Zonas | Exibe solicitações e denúncias no mapa |
| RF08 | Coletor Registrar Coleta | Atualização do status em tempo real |
| RF09 | Módulo de Comunicação | Avisos e notificações ao usuário |
| RF10 | Gestão de Usuários | Função exclusiva do Gestor |
| RF11 | Relatórios e Estatísticas | Dados gerenciais para tomada de decisões |

---

### Requisitos Não Funcionais (RNF)
Os requisitos não funcionais (RNF) estabelecem os critérios de qualidade e as restrições técnicas que devem ser observadas durante o desenvolvimento do sistema, influenciando diretamente a experiência do usuário e a robustez da aplicação. A seguir, cada requisito não funcional é descrito com sua justificativa e importância técnica.

| Código | Nome | Descrição |
|--------|------|-----------|
| RNF01 | Interface Responsiva | Adaptável a smartphone e web |
| RNF02 | Segurança e Criptografia | Proteção de dados do usuário |
| RNF03 | Tempo de Resposta ≤ 5s | Agilidade na experiência |
| RNF04 | Disponibilidade ≥ 90% | Alta confiabilidade |
| RNF05 | Compatibilidade Android e Web | Fácil acesso em diversos dispositivos |
| RNF06 | Armazenamento Seguro | Evitar perda de dados |
| RNF07 | Usabilidade Intuitiva | Para usuários com baixa alfabetização digital |
| RNF08 | Acessibilidade | A interface deve seguir boas práticas de acessibilidade (como WCAG), |
